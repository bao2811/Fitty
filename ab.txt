Detailed UI Design for the Fitty Mobile App

Below is a high-detail screen design specification for the Fitty mobile fitness application. The goal is to describe each interface like a real mobile product, with attention to:

screen purpose
top-to-bottom layout
exact UI components
button behavior
information hierarchy
mobile-friendly interaction patterns
personalization and motivation features

This version is especially strong for UI/UX coursework, Android Compose implementation, or mobile app product design documentation.

1. Overall design principles of Fitty

Fitty is an AI-powered fitness and wellness app, so the user interface should communicate four key feelings:

1. Energy

The app should feel active, healthy, and motivating. The user should feel encouraged to take action every time they open it.

2. Personalization

The app should not look generic. It should feel like a personal coach that understands the user’s goals, habits, progress, and daily condition.

3. Clarity

Users should immediately understand:

what they need to do today
how they are progressing
what functions are most important
where to go next
4. Progress visibility

The user should always be able to see visible progress:

streak
tasks completed
calorie tracking
body changes
achievements
workout completion
2. Main navigation structure

The bottom navigation bar should contain 5 core tabs:

Home
Plan
Track
Coach
Profile
Bottom navigation design
icon on top
text label below
active tab highlighted in brand color
inactive tabs in neutral gray
clean, rounded icons with one consistent style
Home is the default landing tab after login

This structure makes the app feel modern and easy to use, while separating daily dashboard, workout planning, tracking, AI support, and personal account settings clearly.

3. Detailed Home screen design

The Home screen is the most important screen in the app. It is the first place users open every day, so it must answer these questions immediately:

What should I do today?
How am I doing so far?
What is still unfinished?
What can I do quickly from here?
3.1. Purpose of the Home screen

The Home screen should not be just a dashboard. It should function as:

a personalized daily overview
a task-oriented fitness dashboard
a motivation screen
a quick-entry point to all core actions
a reminder system for daily health actions
3.2. Overall Home screen layout from top to bottom
A. Top App Bar

This is the topmost part of the screen and should remain visually stable.

Left side
circular user avatar
greeting text beside it:
Good Morning, Anna
smaller supporting line:
Let’s make today count
Right side
notification bell icon
checklist or calendar icon
optional red badge if there are unread reminders
Interaction
tap avatar → open Profile
tap notification bell → open Notification Center
tap checklist/calendar icon → open Today’s Tasks full list
Design notes
height around 64–72dp
clean background
enough padding on left and right
user name limited to one line
icons medium-sized and easy to tap
B. Hero Summary Card

Users should not have to search for their next action. This card acts as the clear entry point for the day.

Phần này sẽ làm app Fitty mạnh hơn rất nhiều vì không chỉ phụ thuộc vào plan có sẵn.

Mục tiêu:

người dùng có thể tự tạo workout
tự sắp xếp bài tập
tự lên lịch
chỉnh số reps / sets / thời gian
xây plan cá nhân hóa

Khu vực này có thể đặt trong Plan với nút:

Create My Plan
hoặc
Build My Workout

C. KẾT HỢP BÀI TẬP MẪU VÀ PLAN TỰ TẠO

Để app logic hơn, nên cho phép người dùng:

bắt đầu từ program mẫu
rồi customize
hoặc tạo plan hoàn toàn mới

Ví dụ trong Program Detail có nút:

Use as Template

Sau đó app copy program mẫu sang chế độ chỉnh sửa.

Đây là cách rất hợp lý vì:

người mới không bị trống
người có kinh nghiệm vẫn có tự do chỉnh
